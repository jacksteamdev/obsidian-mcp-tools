## 2. Setup ambiente sul nuovo Mac

### Prerequisiti

```bash
# Bun (runtime + package manager)
curl -fsSL https://bun.sh/install | bash

# Git (di solito già presente, eventualmente brew install git)

# GitHub auth — scegli uno dei due:
#   (a) gh CLI con login interattivo
brew install gh && gh auth login
#   (b) SSH key esistente già caricata su github.com

# Obsidian app
# Scarica da https://obsidian.md o `brew install --cask obsidian`
```

### Clone del fork

```bash
# In una directory di lavoro a tua scelta — esempio:
mkdir -p ~/Documents/Projects/Obsidian\ MCP
cd ~/Documents/Projects/Obsidian\ MCP

# HTTPS (richiede gh login o token):
gh repo clone istefox/obsidian-mcp-tools

# Oppure SSH se hai la chiave configurata:
git clone git@github.com:istefox/obsidian-mcp-tools.git

cd obsidian-mcp-tools
```

### Setup remote

Il repo è clonato dalla fork, quindi `origin` punta a `istefox/...`.
Per coerenza con il workflow attuale, rinomina/aggiungi il remote
come `myfork`:

```bash
git remote rename origin myfork
# Opzionale — se vuoi seguire upstream:
git remote add upstream https://github.com/jacksteamdev/obsidian-mcp-tools.git
git fetch --all
```

Verifica con `git remote -v`. L'output deve mostrare `myfork` (tuo
fork, push abilitato) e opzionalmente `upstream` (read-only,
dormant).

### Install dipendenze

```bash
bun install                # installa workspace (server + plugin + shared)
```

### Verifica che tutto funzioni

```bash
bun run check              # type-check su tutti i package
cd packages/obsidian-plugin && bun test && cd ../..
cd packages/mcp-server && bun test && cd ../..
```

Aspettative: **219 test in totale, 0 failure**, type check verde.

### Build una tantum (per esercitare il path)

```bash
cd packages/obsidian-plugin && bun run build && cd ../..
cd packages/mcp-server && bun run build && cd ../..
```

Aspettative: plugin produce `main.js` in radice del repo (non in
`dist/`), server produce `packages/mcp-server/dist/mcp-server` (60
MB, gitignored).

## 3. Setup del vault TEST per integration manuale

Le sessioni precedenti hanno usato un vault Obsidian dedicato per
test manuali a `/Users/stefanoferri/Obsidian/TEST` sul Mac di casa.
Sul Mac dell'ufficio devi ricrearlo:

1. Apri Obsidian → **Create new vault** → chiamalo `TEST` in una
   directory a piacere (p.es. `~/Obsidian/TEST`).
2. Settings → Community plugins → **Turn on community plugins**.
3. Browse → cerca **Local REST API** di Adam Coddington → Install →
   Enable.
4. Settings → Local REST API → verifica che ci sia una API key
   generata (te la appunti, ti serve per le curl di test).
5. Symlink il plugin di sviluppo nel vault:

```bash
cd /path/to/your/checkout/of/obsidian-mcp-tools
mkdir -p ~/Obsidian/TEST/.obsidian/plugins
ln -s "$PWD" ~/Obsidian/TEST/.obsidian/plugins/mcp-tools
```

6. In Obsidian: Settings → Community plugins → attiva **MCP Tools**.
7. Settings → MCP Tools → "Install server" se vuoi anche testare il
   server end-to-end (non strettamente necessario per il lavoro su
   #29 Fase 3).

### Per testare il command-permission gate via curl

Esempio dalla sessione di Fase 2 (sostituire la API key con la tua):

```bash
curl -sk -X POST "https://127.0.0.1:27124/mcp-tools/command-permission/" \
  -H "Authorization: Bearer YOUR_API_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{"commandId":"editor:toggle-bold"}'
```
